# 单票系统字段与模块总契约表 V1

## 0 总则
- 版本：V1
- 主键：`symbol + trade_date`
- 时间格式：`YYYY-MM-DD`
- 字段契约：定义冻结后不得漂移
- 数据层只输出：标准化字段 + 数据质量状态

## 1 数据层（Data Layer）
- L0 raw：行情/指数/事件/公司行为（原样落库）
- L1 clean：统一字段、类型、日期、单位
- L2 base：收益率、波动、价差等基础衍生
- 必备字段：symbol、market、asset_type、trade_date、open/high/low/close、volume、amount
- 质量字段：quality_flag、data_source、data_version、ingest_time

## 2 标签层（Label Layer）
- 目标：统一y定义、避免未来函数污染
- 回归标签：y_close、y_ret；horizon：T+1/T+5/T+20/T+120
- 分类标签：上涨/下跌/大涨/大跌/波动放大/regime切换
- 分位标签：用于 P05~P95 与区间覆盖校准

## 3 特征层（Feature Layer）
- 分类：价格趋势、波动、量价、市场联动、事件窗口
- 元信息：字段名、定义、来源、缺失处理、标准化、版本
- 非交易日不生成虚假bar

## 4 工程接口（Engineering Contracts）
- 文件层级统一：data_raw/data_clean/data_base/data_context/data_events
- 输出到训练层必须带版本：data_version + feature_version + quality_flag
- 任何修订必须可追溯：原因/范围/前后差异
